

# **Project Title: The Most Overused Dad Jokes**  

## **Project Overview:**  
Dad jokes are **everywhere**—some are funny, but most are so **overused** that they should be **retired forever**. This project is a **community-driven platform** where users can **submit, rate, and comment** on classic dad jokes. The goal is to find and officially declare the **most cringe-worthy dad joke of all time**.  

Users will be able to **upvote or downvote** jokes to decide whether they deserve a place on the **Dad Joke Hall of Fame or the Retirement List**. A **random dad joke generator** will provide endless amusement (or suffering), and a **leaderboard** will showcase the best (or worst) dad jokes, ranked by user votes.  

## **Key Features:**  
 **User-submitted dad jokes** – Share your favorite (or least favorite) dad jokes.  
 **Voting system** – Users vote to **“retire”** jokes or crown them as all-time classics.  
 **Random dad joke generator** – Pulls jokes from **JokeAPI** for instant fun.  
 **Leaderboard** – Displays the **best and worst dad jokes**, ranked by votes.  
 **Comment section** – React with groans, laughs, or facepalms.  

## **Tech Stack:**  
 **Frontend:** React (Vite)  
 **Backend:** Node.js, Express.js  
 **Database:** MongoDB (Mongoose)  
 **Random Joke Generator:** JokeAPI integration  
 **Deployment:** Vercel (Frontend), Render (Backend)  

## **Why This Project?**  
🔹 **Fun & interactive** – Encourages user engagement through joke submissions and voting.  
🔹 **API Integration** – Teaches how to work with **external APIs** like JokeAPI.  
🔹 **Covers key full-stack concepts** – Includes **authentication, CRUD operations, database interactions, and ranking systems**.  
🔹 **Scalable** – Can be expanded with features like **user profiles, trending jokes, or joke categories**.
